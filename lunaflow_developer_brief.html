What's Changed

SDK 54 + Modern Expo Router (app/ structure)

This brief has been updated to use Expo Router v3+ with the modern app/ directory structure (file-based routing), replacing the older src/navigation pattern. This matches Expo's current best practices and your existing project layout.

📌 Key Differences from Original Brief

  • Routing: File-based routing via app/ directory instead of RootNavigator.tsx
  • Navigation: expo-router handles tabs automatically via app/(tabs)/ convention
  • Entry point: app/_layout.tsx replaces the top-level navigator setup
  • No more manual bottom-tabs navigation: Layout files define the structure declaratively
  • Easier nested routes: Create new routes by adding files in the app/ directory

Routing Setup

Expo Router Configuration for Modern app/

app/_layout.tsx — Root layout
import { Stack } from 'expo-router'; import { ThemeProvider } from './theme/ThemeContext'; export default function RootLayout() { return ( <ThemeProvider> <Stack screenOptions={{ headerShown: false }} /> </ThemeProvider> ); }
app/(tabs)/_layout.tsx — Tabs navigator
import { Tabs } from 'expo-router'; import { colors } from '@/theme/colors'; export default function TabsLayout() { return ( <Tabs screenOptions={{ tabBarActiveTintColor: colors.primary, tabBarInactiveTintColor: colors.muted, tabBarStyle: {{ backgroundColor: colors.surface }}, headerShown: false, }} > <Tabs.Screen name="index" options={{ title: 'Home' }} /> <Tabs.Screen name="log" options={{ title: 'Log' }} /> <Tabs.Screen name="calendar" options={{ title: 'Calendar' }} /> <Tabs.Screen name="insights" options={{ title: 'Insights' }} /> <Tabs.Screen name="settings" options={{ title: 'Settings' }} /> </Tabs> ); }
app/(tabs)/index.tsx — Home Screen file
import { View, ScrollView } from 'react-native'; import { StatusCard, CycleRing, ComingUpCard, QuickLog } from '@/components/screens/home'; export default function HomeScreen() { return ( <ScrollView style={{ flex: 1, backgroundColor: '#0A0B0F' }}> <StatusCard /> <CycleRing /> <ComingUpCard /> <QuickLog /> </ScrollView> ); }

Why this structure? Each `.tsx` file in app/(tabs)/ becomes a tab route. No more manual navigation setup. Expo Router handles it.

Data Layer

Zustand + AsyncStorage Persistence

store/cycleStore.ts
import { create } from 'zustand'; import { persist, createJSONStorage } from 'zustand/middleware'; import AsyncStorage from '@react-native-async-storage/async-storage'; import type { CycleState, Cycle } from '@/types/cycle.types'; const DEFAULT_CYCLE: Cycle = { lastPeriodDate: '', cycleLength: 28, periodLength: 5, }; export const useCycleStore = create<CycleState>( persist( (set) => ({ cycle: DEFAULT_CYCLE, setCycle: (cycle) => set({ cycle }), resetCycle: () => set({ cycle: DEFAULT_CYCLE }), }), { name: 'lunaflow-cycle', storage: createJSONStorage(() => AsyncStorage), } ) );
store/logStore.ts
import { create } from 'zustand'; import { persist, createJSONStorage } from 'zustand/middleware'; import AsyncStorage from '@react-native-async-storage/async-storage'; import type { LogStore, LogEntry } from '@/types/log.types'; export const useLogStore = create<LogStore>( persist( (set, get) => ({ logs: {}, addLog: (entry: LogEntry) => set((state) => ({ logs: { ...state.logs, [entry.date]: entry }, })), getLog: (date: string) => get().logs[date], removeLog: (date: string) => set((state) => {{ logs: Object.fromEntries( Object.entries(state.logs).filter(([k]) => k !== date) ), }}), }), { name: 'lunaflow-logs', storage: createJSONStorage(() => AsyncStorage), } ) );
Business Logic

Cycle Calculation Engine (cycleCalc.ts)

This file contains the **heart** of the app. All cycle phase logic, day calculations, and predictions happen here. Must be unit tested.

utils/cycleCalc.ts — Core exports
import type { DayType, Phase } from '@/types/cycle.types'; // getCycleDay(date: string, lastPeriodDate: string): number // Returns 1–28 (or longer). Which day of current cycle is this? // getPhase(cycleDay: number): Phase // Returns phase based on day: menstrual (1–5), follicular (6–13), ovulation (14–15), luteal (16–28) // getDayType(cycleDay: number, hasPeriod: boolean): DayType // Returns color type: 'period' | 'fertile' | 'ovulation' | 'normal' | 'future' // getNextPeriod(cycleDay: number, cycleLength: number): number // Days until next period starts // getPhaseProgress(cycleDay: number, phase: Phase): { current: number; total: number } // How far through the current phase? Used for phase timeline progress bar

  • Test when lastPeriodDate is in the future (should return 0 or handle gracefully)
